# Chapter 05 - Literals

This chapter explains literal values in JavaScript: numbers, strings, `null`, `undefined`, and template literals.

## Practical Files

| File | Practical | What happens |
| --- | --- | --- |
| `17_literals.js` | Literal basics | Introduces direct values written in code. |
| `18_null_undefined.js` | `null` and `undefined` | Shows the difference between empty value and not assigned. |
| `19_Literals_ALL.js` | All literal types | Collects multiple literal examples together. |
| `20_literals_number_all.js` | Number literals | Practices numeric values and number formats. |
| `21_StringLiterals.js` | String literals | Shows strings with quotes. |
| `22_Templet_Literals.js` | Template literals | Uses backticks and interpolation. |
| `23_Backtick_single_double.js` | Quote comparison | Compares single quotes, double quotes, and backticks. |

## Concepts Covered

- Number literals
- String literals
- Template literals with `${...}`
- `null` vs `undefined`
- Choosing quote styles

## How To Run

```bash
node Chapter_05_literals/17_literals.js
node Chapter_05_literals/22_Templet_Literals.js
```

## Takeaway

Literals are the raw values your program works with. Template literals make dynamic strings easier to read.
