# Chapter 08 - Switch

This chapter covers `switch` statements and how they compare with if/else chains. It includes default cases, missing break behavior, grouped cases, and real examples.

## Practical Files

| File | Practical | What happens |
| --- | --- | --- |
| `Switch.js` | Basic switch | Matches one value against multiple cases. |
| `Default.js` | Default case | Handles unmatched values. |
| `No_Break.js` | Missing `break` | Demonstrates fall-through behavior. |
| `Switch_Group.js` | Grouped cases | Shares one result across multiple cases. |
| `REAL_TIME_EXAMPLE.js` | Real example | Applies switch to practical category logic. |
| `IQ.js`, `IQ2.js`, `IQ3.js`, `IQ4.js` | Interview questions | Practices switch output and edge cases. |

## Concepts Covered

- `switch`, `case`, `break`, and `default`
- Fall-through behavior
- Grouping cases
- When switch is cleaner than many `else if` blocks

## How To Run

```bash
node Chapter_08_Switch/Switch.js
node Chapter_08_Switch/No_Break.js
node Chapter_08_Switch/Switch_Group.js
```

## Takeaway

Use `switch` when one value needs to be matched against many clear options.
